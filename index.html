Перелік основних команд

  • config --global user.name blabla - введення ім*я користувача. // вхід в обліковий запис git
  • config --global user.email blabal@bla - введення е-мейла користувача.// вхід в обліковий запис git
  • git init - призначена для створення пустої папки репозиторію.
  • git status - команда показує файли, які раніше не відслідковувались в каталозі.
  • git add - команда виконує індексації для по відслідковуванні подальших змін у файлі.
  • git commit - виконується для фіксації змін.
  • git log - відображає історію зафіксованих змін (бек-апів).
  • git checkout - команда для переходу між зафіксованими змінами там здійснення переходу на інший branch.
  • git branch - при запуску без параметрів - виводить список доступних гілок (branch).
  • git push - відправляє збережені зміни на віддалений репозиторій. git push origin master - відправляє оновлення гілки branch master насервер origin.
  • git pull - підтягує останні вснесені змінні з віддаленого репозиторію.
  • git merge - виконує злиття гілочок branch.
  • git clone - копіює дані з віддаленого репозиторію на локальний, використовується - якщо вже хтось вніс змінні і виконав команду push на віддалений репозиторій.
  • git revert - використовується для скасування коміту, для реалізації потрібно створити новий коміт, який буде замість небажаного коміта.
  • git reset - дозволяє знести останні зміни у гілці branch, працює в різних режимах та з різними параметрами, починаючи під відкату до попередньої версії і повного видалення текучої - до відкочення до попередньої і надання їй статусу HEAD.
  • git rebase - виконує злиття гілочок branch зі зміною хешування. Всі коміти зливаються в одну гілку послідовно
  • git cherry-pick - Використовується для злиття декількох змін в новий комміт, який матиме статус HEAD - дозволяє використовувати комміти з інших гілочок branch без їхнього злиття.
  • git stash - відслідковує та приховує всі зміни в індексованих файлах.
  • .gitignore - використовується для ігнорування файлів при індексаціях, у каталозі /Config створюється текстовий документ .gitignore з вмістом **/README.txt, це дозволить робити подальші комміти без файлуREADME.txt
